9. PROMPT: "the two versions ... are dealing with system time differently. The
   displayed positions are offset by 1 hr ... applies to all dates from March 9 to
   Nov. 1, so something about how daylight savings is addressed."
   ACTION: Fixed a DST parity bug. The original AS sets currentYear = now.getYear()
   (year MINUS 1900, e.g. 126) and feeds it back into new Date(126, ...), i.e. dates
   in year 126 AD where no DST exists -- so its daylightSavingDelta is 0 and the sky
   is NEVER DST-shifted. An earlier version of this port had "corrected" that with
   the real year + real DST delta, which applied a genuine -1 hour shift across the
   DST window (~Mar-Nov) and put the stars 1 hour (~15 deg) off from the original.
   Fix in simulation.js update(): the sky now uses
     solDaysSinceZero = dayOfYear + timeOfDay - 78.5   (NO DST offset),
   matching the original exactly. The "daylight saving time in effect" placard is
   kept (informational) and still uses the real current year so it shows only on true
   DST dates. Verified: rotation now +15.04 deg vs the buggy value (= one hour),
   orientation matches the original's left screenshot; no console errors.
   Doc: CONVERSION_NOTES.md deviation #3 rewritten.

    - Verified via the Browser pane (javascript_tool): clock focusable, all keys work

    Files: index.html (clock canvas role/aria), simulation.js (clockKey, stepHour/

RECURRING NOTE -- LOCAL DEV CACHING
================================================================================
The local "python -m http.server" sends no cache-control headers, so browsers may
keep serving a cached simulation.js / styles.css after edits. If a change does not
appear, HARD-REFRESH: Ctrl + Shift + R (Windows/Linux) or Cmd + Shift + R (macOS),
or open DevTools -> Network -> "Disable cache". This is a local-dev artifact only;
the production cloud host serves proper cache headers.

All changes above are confined to index.html, styles/styles.css, and simulation.js.
The KL-UNL foundation files remain unchanged except for the one documented JSON
syntax repair to contents.json (see CONVERSION_NOTES.md).
